### Rules

- **R-PROTO-001** MUST: Implement protocol-based public API design pattern that defines explicit contracts for external interactions.
- **R-PROTO-002** MUST: Establish formal protocols/interfaces that serve as the public API surface, separating the contract from implementation details.
- **R-PROTO-003** MUST: Apply the pattern consistently across integration test scenarios (no tailor, sync modes, smoke tests).
- **R-PROTO-004** MUST: Ensure external consumers interact through well-defined protocol boundaries rather than directly with internal implementations.
- **R-PROTO-005** MUST: Maintain clear separation between public API contracts and internal implementation to enable independent evolution.
- **R-PROTO-006** SHOULD: Design integration tests that validate external-facing behavior through protocol boundaries.
- **R-PROTO-007** SHOULD: Maintain protocol boundaries and avoid leaking implementation details.
- **R-PROTO-008** SHOULD: Use protocols to explicitly define the public contract for documentation purposes.
- **R-PROTO-009** MAY: Accept additional abstraction layer complexity as trade-off for protocol separation.

### Verify

**Note:** The source ADR does not include explicit verification commands. Verification should focus on:
- Confirming protocol/interface definitions exist for public API surface
- Validating that integration tests interact through protocols, not implementations
- Checking that implementation details are not exposed through public contracts

**Accept when:**
- Public API contracts are defined as explicit protocols/interfaces separate from implementation
- Integration tests across all scenarios (no tailor, sync modes, smoke tests) use protocol-based boundaries
- External consumers cannot directly access internal implementation details
- Protocol boundaries provide clear separation enabling independent evolution
- Documentation surface clearly reflects the public contract through protocols
- Backward compatibility guarantees are maintained for external API consumers

<enforcement>
Claude Code MUST verify protocol-based design patterns are maintained when modifying test files. Protocol boundaries must not be bypassed in integration tests.
</enforcement>
